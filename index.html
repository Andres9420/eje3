Raspberry como Sistema Embebido

¿Qué es un Sistema Embebido?

Un sistema embebido es un dispositivo electrónico que integra hardware y software diseñados para cumplir una función específica dentro de un sistema mayor. Aunque no se perciba como una computadora tradicional, sigue el mismo esquema de entrada, procesamiento y salida. Está presente en dispositivos cotidianos como microondas, ascensores o televisores, respondiendo de forma automatizada a acciones simples como presionar un botón..

1. Características y Componentes

Incluyen procesador, memoria, interfaces de entrada/salida y un sistema operativo específico para su función. Se enfocan en eficiencia, bajo consumo y estabilidad.

Características de los sistemas embebidos

  • Realizan funciones específicas: Están diseñados para llevar a cabo tareas altamente especializadas.
  • Funcionan prolongadamente: Están diseñados para operar de manera continua durante largos períodos de tiempo sin interrupciones. Esta facultad es esencial para muchas aplicaciones.
  • Inician rápidamente: Tienen la capacidad para arrancar de manera rápida y estar listos para funcionar en un tiempo mínimo. La rapidez con la que un sistema embebido puede ponerse en marcha es vital en situaciones donde se necesita una respuesta inmediata.
  • Son eficientes energéticamente: Están optimizados para consumir la menor cantidad de energía posible. Es importante sobre todo en aplicaciones que funcionan con baterías o fuentes de energía limitadas.
  • Destacan por su confiabilidad: La fiabilidad es una característica clave de los sistemas embebidos, ya que se espera que funcionen sin fallos en entornos críticos (como los relacionados a la salud o la seguridad).
  • Es limitado en recursos: Suelen contar con recursos limitados en términos de memoria, capacidad de procesamiento y almacenamiento.

Componentes de un sistema embebido

A grandes rasgos, un sistema embebido está compuesto por lo siguiente:

  • Microprocesador: Este componente actúa como el núcleo del sistema. Es un chip diseñado específicamente para tareas embebidas. Controla todas las operaciones, ejecuta instrucciones y gestiona los recursos del sistema de manera eficiente.
  • Memoria: Hay dos tipos de memoria en un sistema embebido. La memoria RAM (Random Access Memory) se utiliza para almacenar datos temporales y variables de trabajo. La memoria flash, por otro lado, almacena programas y datos permanentes, como el software del sistema y la configuración.
  • Periféricos: Son dispositivos adicionales conectados al sistema embebido, los cuales permiten la interacción con el entorno. Abarcan elementos como sensores para recopilar datos del entorno, actuadores para realizar acciones físicas, y dispositivos de entrada/salida para interactuar con los usuarios.
  • Software: Es la parte inteligente del sistema. Comprende el conjunto de programas y algoritmos que definen las funciones específicas del sistema embebido. Este software dicta cómo el sistema responde y se comporta.
  • Fuente de poder: Proporciona la energía necesaria para que el sistema funcione correctamente. Puede ser una batería, un adaptador de corriente o cualquier otra fuente de energía diseñada para el sistema en particular.
  • Circuito de reloj: Es el responsable de sincronizar las operaciones del sistema. Define el ritmo al que el microprocesador ejecuta instrucciones y coordina el funcionamiento de todos los componentes para garantizar un rendimiento apropiado.

2. Vulnerabilidades

Vulnerabilidades de Software

  • Desbordamientos de búfer
  • Ataques de intermediario
  • Ataques de denegación de servicio (DoS)
  • Inyección de código
  • Falta de validación de entradas
  • Falta de seguridad en el almacenamiento de contraseñas

Vulnerabilidades de Hardware

  • Ataques a la red
  • Ataques físicos
  • Vulnerabilidades de hardware

Vulnerabilidades específicas de los sistemas embebidos

  • Recursos limitados
  • Restricciones energéticas
  • Falta de actualizaciones

3. Arquitectura de Hardware

Estructura común del hardware en sistemas embebidos

Aunque cada sistema embebido puede tener una arquitectura distinta según su función, muchos comparten una estructura básica como la siguiente:

  • Procesador
    • Microprocesador
    • Microcontrolador de 4, 8, 16 y 32 bits
    • Procesador de Señales Digitales (DSP)
    • Matriz de puertas programables
  • Dispositivo de almacenamiento
  • Interfaces de comunicación
  • Puertos para transmisión de datos


4. Dispositivo Embebido: Raspberry Pi

La Raspberry Pi es una computadora de bajo costo y tamaño reducido, creada para enseñar computación, pero que ha sido ampliamente adoptada en el desarrollo de proyectos embebidos. Aunque no es un sistema embebido puro, como un microcontrolador dedicado, su capacidad para ejecutar Linux, controlar hardware a través de puertos GPIO y adaptarse a tareas específicas la convierte en una plataforma ideal para aplicaciones como estaciones meteorológicas, domótica, automatización o robótica. Su versatilidad, comunidad activa y facilidad de uso la hacen una herramienta accesible tanto para el aprendizaje como para la creación de sistemas embebidos funcionales.

4.2 Especificaciones Tecnicas Raspberry pi 4

  • Procesador: Broadcom BCM2711, Cortex-A72 de cuatro núcleos (ARM v8) SoC de 64 bits a 1,5 GHz
  • Memoria: 8GB LPDDR4
  • Conectividad:
    • Wi-Fi IEEE 802.11 b/g/n/ac (2.4 GHz y 5.0 GHz)
    • Bluetooth 5.0, BLE
    • Gigabit Ethernet
  • Puertos USB:
    • 2 x USB 3.0
    • 2 x USB 2.0
  • GPIO: Estándar de 40 pines
  • Video y sonido:
    • 2 puertos micro HDMI (hasta 4Kp60 soportado)
    • Puerto DSI para pantalla
    • Puerto CSI para cámara
  • Almacenamiento: Ranura para tarjeta microSD (para sistema operativo y datos)
  • Alimentación:
    • 5V DC a través del conector USB-C (mínimo 3A)
    • 5V DC a través de los GPIO (mínimo 3A)
    • Power Over Ethernet (PoE), mediante HAT PoE adicional
  • Temperatura de funcionamiento: 0–50 °C
  • Dimensiones: 88 x 58 x 18.5 mm
